## Three preflight modes

`preflight()` is the SDK's one-call gate that an FE can wrap around any
outgoing call before signing. It returns `{ ok, reason, reasonText, source }`
and **never throws on a policy rejection** — only on programmer error.

| Mode | When to use | RPC? | Source of truth |
|------|-------------|------|-----------------|
| `chain` | Production. Live on-chain state matters and policies change. | yes | `WardOracle.checkIntent` |
| `local` | Cached / hot path. Policy already loaded into an `EvalPolicy`. | no | in-memory `EvalPolicy` |
| `spec`  | FE bundles a `POLICY.md`; operator owns the file, not on-chain state. | no | the YAML doc |

```ts
import { preflight, evalPolicyFromInput, compilePolicy } from "@ward/sdk";

// Mode A — chain
await preflight({
  source: { kind: "chain", publicClient, oracleAddress, policyId },
  intent,
  spentTodayWei,
});

// Mode B — local (pre-compiled policy)
const policy = evalPolicyFromInput(compilePolicy(policyMd));
await preflight({ source: { kind: "local", policy }, intent, spentTodayWei });

// Mode C — spec (compile + evaluate in one shot)
await preflight({ source: { kind: "spec", yaml: policyMd }, intent, spentTodayWei });
```

The TS evaluator (`evalCheckIntent`) is a bit-faithful port of
`PolicyLib.validate` + the tier branch in `WardOracle.checkIntent`. Parity
is enforced by `sdk/tests/policy-eval.parity.test.ts`, which replays a JSON
fixture generated by `contracts/test/PolicyEvalFixtures.t.sol` — any drift
between Solidity + TS surfaces as a failing test, not silent disagreement.

## Read a policy from an agent

```ts
import { createPublicClient, http } from "viem";
import { createOracleClient } from "@ward/sdk";

const oracle = createOracleClient({
  publicClient: createPublicClient({ transport: http("https://dream-rpc.somnia.network") }),
  // v2 oracle (canonical for new integrations; v0.11.0+).
  // Pass "0x68d4B045B24F8d1012974b9d34684cA5aeD11DDf" if you're reading a pre-v0.11.0 policy on the v1 oracle.
  oracleAddress: "0x3C7bF90f243d670a01f512221d9546e09fEaCC9c",
});

const { ok, reason } = await oracle.checkIntent(policyId, intent, spentToday);
```

`checkIntent` is a read; `publishPolicy` / `updatePolicy` require passing a `walletClient`.

## Find Ward-watched agents

Other agents and services on Somnia can discover agents protected by Ward by reading the on-chain WardAgentRegistry directly — no indexer required.

```ts
import { createPublicClient, http } from 'viem';
import { findWardAgents } from '@ward/sdk';

const somnia = {
  id: 50312,
  name: 'Somnia Shannon Testnet',
  nativeCurrency: { name: 'STT', symbol: 'STT', decimals: 18 },
  rpcUrls: { default: { http: ['https://dream-rpc.somnia.network'] } },
} as const;

const publicClient = createPublicClient({ chain: somnia, transport: http(somnia.rpcUrls.default.http[0], { timeout: 8_000 }) });

const result = await findWardAgents({
  publicClient,
  registryAddress: '0x97F743A9AAa5AcAA73075C1B8F1921274755CF70',
  onlyActive: true,
});

if (result.ok) {
  for (const a of result.agents) {
    console.log(a.name, a.agent, a.tags, 'policy=', a.policyId);
  }
} else {
  // result.agents is undefined when ok:false; partial results may still exist.
  const partial = result.agents ?? [];
  console.warn('discovery error:', result.error, 'partial:', partial.length);
}
```

Returns a `Result` envelope so a single failed page doesn't lose the rest:

| field | type | meaning |
|---|---|---|
| `ok` | `boolean` | true if all pages decoded cleanly |
| `agents` | `WardAgent[]` | every agent successfully decoded (may be non-empty even when `ok=false`) |
| `totalCount` | `bigint` | snapshot of `agentCount()` at the start of the walk |
| `pagesRead` | `number` | how many `agentsPaginated` reads succeeded |
| `error` | `string?` | reason the walk stopped, when `ok=false` |

The registry is permissionless — anyone can register their own agent. Tags are free-form strings the registrar attaches; treat them as untrusted.

## Gate a frontend with Ward preflight

Run a Ward policy preflight in the browser so the user sees why a call would revert before the wallet popup opens.

### When to preflight

Preflight when you want fast UX feedback on a state-changing call: surface the
policy decision early, skip a wallet prompt that would only revert, and show the
user the reason code.

Preflight is a UX layer, **not a security boundary**. The on-chain `WardOracle`
still enforces the policy and the agent contract still reverts when the calldata
does not match. If a user bypasses your UI and calls the agent directly, the
contract reverts on the same policy. Treat the preflight as a confirmation, not
a gate. The browser-side decision is advisory; the on-chain check is
authoritative.

### Pick a source kind

Every preflight entry point — the SDK's `preflight()`, the React hooks, and the
Vite plugin's output — selects from the same three source kinds (see the
[Three preflight modes](#three-preflight-modes) table above). Pick the one
that matches where your policy truth lives.


### `useWardGuardedWrite`

Pre-flights every write against the policy. If the policy rejects, the hook
throws `WardPreflightRejectedError` with a decoded reason and no wallet
prompt is shown.

```tsx
import { useWardGuardedWrite } from "@ward/react";
import { agentAbi } from "./abi";

function BumpButton({ agent, oracle, policyId, publicClient }) {
  const { write, isPreflightPending, lastDecision } = useWardGuardedWrite({
    source: { kind: "chain", publicClient, oracleAddress: oracle, policyId },
  });

  async function onClick() {
    try {
      await write({
        abi: agentAbi,
        address: agent,
        functionName: "bump",
        args: [1n], // bump(uint256 by)
      });
    } catch (err) {
      // err.decision is the typed PreflightResult when the rejection came
      // from Ward preflight. Surface lastDecision in your UI either way.
    }
  }

  return (
    <button onClick={onClick} disabled={isPreflightPending}>
      {isPreflightPending ? "Checking policy…" : "Call bump"}
    </button>
  );
}
```

#### Handling a rejected decision

On a reject, `write` rejects with `WardPreflightRejectedError`. When the
rejection came from Ward preflight, `err.decision` is the typed
`PreflightResult` — but you should surface `lastDecision` from the hook either
way, since it holds the latest decision regardless of how the promise settled.
Because the rejection short-circuits before `writeContract`, the wallet popup
never opens.

#### Local and spec sources

`source.kind: "local"` swaps the oracle call for an in-memory policy:

```ts
const { write } = useWardGuardedWrite({
  source: { kind: "local", policy: compiledPolicy },
});
```

`source.kind: "spec"` accepts the original `POLICY.md` text and compiles it
once per call (cheap; the result is cached inside `preflight`).

### `useWardActionPlan`

Returns a `WardActionPlan` (`write-now`, `queue`, or `reject`) for a
prepared intent so the UI can show the queue tier and delay before the
user clicks. The hook re-evaluates on `refresh()` and exposes
`isPending` + `error` for loading states.

```tsx
const { plan, refresh } = useWardActionPlan({
  source: { kind: "local", policy },
  intent,
  spentTodayWei: 0n,
});
```

`queue` plans only resolve for `local` and `spec` sources today — the
chain source returns the raw decision and the caller must look up the
tier separately. `useWardActionPlan` raises a typed error if you try
to plan a queue with `kind: "chain"`.

### Honest scope

Same boundary as [When to preflight](#when-to-preflight): this is a UX layer, not a security boundary.

## Vite plugin (`@ward/vite-plugin`)

Vite plugin that turns `POLICY.md?eval-policy` imports into an evaluated
`EvalPolicy` object at build time. The plugin runs the same compiler
the SDK uses (`compilePolicy` → `evalPolicyFromInput`) so the policy
your bundle ships is the same one the contracts validate.

### Status

Lives inside the Ward monorepo and ships as `private: true`. Add it through the workspace.

### Install (workspace)

```jsonc
{
  "devDependencies": {
    "@ward/vite-plugin": "workspace:*",
    "@ward/sdk": "workspace:*",
    "vite": "^5.4.0"
  }
}
```

Then run `pnpm install` at the repo root.

### Wire it into `vite.config.ts`

```ts
import { defineConfig } from "vite";
import wardPolicy from "@ward/vite-plugin";

export default defineConfig({
  plugins: [wardPolicy()],
});
```

### Import a policy file

Append `?eval-policy` to any markdown file containing a policy block:

```ts
import policy from "./POLICY.md?eval-policy";

console.log(policy.dailySpendWeiCap, policy.tier);
```

The plugin parses the markdown, compiles the policy, and emits a
JavaScript module whose default export is the evaluated `EvalPolicy`
(bigint fields serialised as `…n` literals). Pass it straight into the
`local` source for `preflight` or `useWardGuardedWrite`:

```ts
import policy from "./POLICY.md?eval-policy";

useWardGuardedWrite({ source: { kind: "local", policy } });
// or, outside React:
// preflight({ source: { kind: "local", policy }, intent, spentTodayWei });
```

Action plan:

1. Add the plugin to your Vite project.
2. Import your POLICY.md once at app boot.
3. Pass the imported policy into `preflight({ source: { kind: "local", policy } })`
   or `useWardGuardedWrite({ source: { kind: "local", policy } })`.

The plugin owns no runtime — it only transforms `?eval-policy` modules.
Everything else still flows through `@ward/sdk` and your on-chain
oracle.

### Core types

From `src/types.ts`:

```ts
export const TIER_IMMEDIATE = 0 as const;
export const TIER_DELAYED = 1 as const;
export const TIER_VETO_REQUIRED = 2 as const;
export type Tier = typeof TIER_IMMEDIATE | typeof TIER_DELAYED | typeof TIER_VETO_REQUIRED;

export const TIER_NAMES = {
  IMMEDIATE: TIER_IMMEDIATE,
  DELAYED: TIER_DELAYED,
  VETO_REQUIRED: TIER_VETO_REQUIRED,
} as const;
export type TierName = keyof typeof TIER_NAMES;

export interface SelectorRule {
  selector: Hex;          // 4-byte selector, "0x..."
  valueCapPerCall: bigint;
  tier: Tier;
  delaySeconds: number;
}

export interface TargetRule {
  target: Address;
  selectors: SelectorRule[];
}

export interface PolicyInput {
  targets: TargetRule[];
  dailySpendWeiCap: bigint;
  maxSlippageBps: number;
  expiresAt: bigint;       // unix seconds
  paused: boolean;
}

export interface Intent {
  agentId: bigint;
  requestId: bigint;
  target: Address;
  selector: Hex;
  data: Hex;
  value: bigint;
  promptHash: Hex;
  taskClass: number;
}
```

`PolicyInput` is the wire shape passed to `publishPolicy` / `updatePolicy`. `Intent` mirrors the on-chain `struct Intent`. `EvalIntent` (from `policy-eval.js`) is a structurally identical "evaluator input" shape used by the off-chain evaluator and `buildIntent`.

### Policy authoring

Three ways to produce a `PolicyInput`: compile a POLICY.md document, build one fluently, or assemble it by hand. The POLICY.md schema itself is documented in the policy spec section of [SKILL.md](../SKILL.md).

#### compilePolicy

```ts
function compilePolicy(markdown: string, opts?: CompileOptions): PolicyInput
```

Extracts the single fenced ` ```policy ` block (or the only untagged fence) from a markdown string, validates it against the JSON schema, applies the semantic refinements below, and returns a canonical `PolicyInput`. This is the SDK's last-line defense before the contract — the CLI bypasses any dashboard schema, so these checks are mirrored here.

`CompileOptions`:

| Field | Type | Effect |
|---|---|---|
| `fenceTag` | `string` | Fence tag to look for. Default `"policy"`; an untagged block also matches when exactly one is present. |
| `oracleAddress` | `Address` | When set, the compiler rejects any target equal to it (self-gatekeeper rug path). |
| `queueAddress` | `Address` | When set, rejects any target equal to it, same reason. |
| `label` | `string` | When set, rejects control bytes (C0 controls + DEL) in the policy label. |

Throws (programmer/author error) for any of: no fenced block found; multiple untagged blocks; YAML parse error; schema validation failure; a target `name` field (not stored on-chain — describe targets in the policy header); duplicate target (case-insensitive) or duplicate selector within a target; zero-address target; EIP-55 checksum failure on a mixed-case target; reserved target (configured oracle/queue or an EVM precompile `0x01..0x0a`, `0x100`); a missing `valueCapPerCall` on a selector (must be present, use `"0"` to block all native value); `dailySpendWeiCap` or `valueCapPerCall` above `uint256` max; `delaySeconds` above `uint32` max; `expiresAt` equal to `0`, at or before `now + 60s` safety window, above `uint64` max, or more than 5 years out; a malformed function signature; an `IMMEDIATE`/`VETO_REQUIRED` selector that sets `delaySeconds`; a `DELAYED` selector with `delaySeconds <= 0`; a control byte in `label`. There is no Result envelope — `compilePolicy` always throws on bad input.

Selector fields accept either a 4-byte hex selector (`0x12345678`, stored as-is) or a Solidity function signature (`transfer(address,uint256)`, run through viem's `parseAbiItem` + `toFunctionSelector`; a malformed type like `addresss` throws).

`maxSlippageBps` is stored on the policy but is **not** enforced by the oracle's `checkIntent` — there is no slippage reason code. Slippage enforcement is left to call sites.

#### PolicyBuilder

Fluent builder for `PolicyInput` (`src/policy-builder.ts`):

```ts
const policy = new PolicyBuilder()
  .target("0x...USDso")
    .selector("approve(address,uint256)", { perCallCap: "0", tier: "IMMEDIATE" })
  .target("0x...DreamDEX")
    .selector("placeOrder(address,address,uint256,uint256)", { perCallCap: "0", tier: "IMMEDIATE" })
  .dailyCap("1 ether")
  .maxSlippageBps(50)
  .expiresInDays(30)
  .build();
```

Methods (all chainable, return `this`):

| Method | Signature | Notes |
|---|---|---|
| `target` | `(addr: Address)` | Opens a new target; subsequent `.selector()` calls attach to it. |
| `selector` | `(sigOrSelector: string \| Hex, opts?: { perCallCap?: string \| bigint; tier?: TierName; delaySeconds?: number })` | `tier` defaults `"IMMEDIATE"`, `perCallCap` defaults `0n`, `delaySeconds` defaults `0`. Throws if called before `.target()`. |
| `dailyCap` | `(amount: string \| bigint)` | String form goes through `parseEtherFlexible`. |
| `maxSlippageBps` | `(bps: number)` | Throws unless `0..10000`. |
| `expiresAt` | `(unixSeconds: number \| bigint)` | |
| `expiresInDays` | `(days: number)` | Computes `now + days*86400`. |
| `paused` | `(p = true)` | |
| `build` | `(): PolicyInput` | Throws if no targets or `expiresAt` unset. |

Throws on: `.selector()` before `.target()`; `IMMEDIATE`/`VETO_REQUIRED` tier with non-zero `delaySeconds`; `delaySeconds` above `uint32` max; `valueCapPerCall` or `dailySpendWeiCap` above `uint256` max; `maxSlippageBps` out of `0..10000`; `build()` with no targets or unset `expiresAt`. The builder does **not** re-run the past-expiry / DELAYED-positive-delay / reserved-target refinements that `compilePolicy` does — those are POLICY.md-document checks.

#### parse helpers

```ts
function parseEtherFlexible(input: string): bigint
function suggestEtherFix(raw: string): string | null
```

`parseEtherFlexible` accepts `"1 ether"`, `"0.5 ether"`, a plain wei integer (`"1000000000000000000"`), or `0x`-prefixed hex wei. Empty string and `"0"` return `0n`. On a near-miss typo for the `ether` suffix (`"0.5ethe"`, `"1 eth"`) it throws with a "did you mean" hint. STT is the only metered native unit — `gwei`/`wei`/`finney` are deliberately not recognized as valid suffixes (they fall through to a raw `BigInt()` failure, not a hint). `suggestEtherFix` returns the suggested canonical form (`"<amount> ether"`) or `null` when no obvious typo is detected.

### Clients

Thin viem wrappers. Reads work in read-only mode (no `walletClient`); writes require a wallet with an account. Asking agents typically enqueue/dispatch from on-chain code — these clients are for off-chain monitoring, indexing, and operator tooling.

#### createOracleClient

```ts
function createOracleClient(args: CreateOracleClientArgs): OracleClient
function policyIdFor(publisher: Address, label: Hex): Hex   // local keccak256, no RPC

interface CreateOracleClientArgs {
  publicClient: PublicClient;
  walletClient?: WalletClient;
  oracleAddress: Address;
  chainId?: number;        // when set, pins every write to this chain at the wallet boundary
}

interface OracleClient {
  readonly address: Address;
  publishPolicy(label: Hex, input: PolicyInput): Promise<{ txHash: Hex; policyId: Hex }>;
  updatePolicy(policyId: Hex, input: PolicyInput): Promise<{ txHash: Hex }>;
  checkIntent(policyId: Hex, intent: Intent, spentToday: bigint): Promise<{ ok: boolean; reason: Hex }>;
  tierAndDelay(policyId: Hex, target: Address, selector: Hex): Promise<{ tier: number; delaySeconds: number }>;
}
```

`publishPolicy` / `updatePolicy` simulate via `publicClient.simulateContract` before submitting, so a revert surfaces *before* the wallet popup opens. `publishPolicy` also returns the locally-computed `policyId` (= `policyIdFor(walletAccount, label)`). `checkIntent` and `tierAndDelay` are pure reads.

Methods on the oracle ABI that the client does **not** wrap — call them through `publicClient.readContract` / `writeContract` with `WARD_ORACLE_ABI` directly: `acceptPolicyOwnership`, `transferPolicyOwnership`, `cancelPolicyOwnershipTransfer`, `policyOwner`, `pendingPolicyOwner`, `policyVersion`, `policyHealth`, `policyIdFor` (on-chain view — but prefer the exported local `policyIdFor`).

#### createQueueClient

```ts
function createQueueClient(args: CreateQueueClientArgs): QueueClient

interface CreateQueueClientArgs {
  publicClient: PublicClient;
  walletClient?: WalletClient;
  queueAddress: Address;
}

type QueueState = "None" | "Pending" | "Committed" | "Vetoed" | "Expired";

interface QueueClient {
  readonly address: Address;
  enqueue(policyId: Hex, intent: Intent, spentToday: bigint): Promise<{ txHash: Hex }>;
  dispatch(execId: bigint): Promise<{ txHash: Hex }>;
  veto(execId: bigint, reason: Hex): Promise<{ txHash: Hex }>;
  expireIfStale(execId: bigint): Promise<{ txHash: Hex }>;
  getRecord(execId: bigint): Promise<QueueRecord>;
  getRecordHeader(execId: bigint): Promise<QueueRecordHeader>;
  nextExecId(): Promise<bigint>;
}
```

`getRecord` returns the full `QueueRecord` (includes the embedded `intent`); `getRecordHeader` returns the lighter `QueueRecordHeader` (adds `target`, `selector`, `value`, `requestId` but drops `intent`). Both decode the `state` enum to its `QueueState` string name.

`getRecordHeader` transparently handles the legacy v1 queue (`0x98A3…90D5`), whose `RecordHeader` predates the `policyVersion uint64` field: it retries the decode with `LEGACY_WARD_QUEUE_ABI_V0` and synthesizes `policyVersion: 0n`. If both the canonical (384-byte) and legacy (352-byte) decodes fail it throws an error naming the queue address and both decode errors.

Queue ABI functions the client does not wrap (call via raw ABI): `oracle()`, `COMMIT_WINDOW_SECONDS()`. Note `enqueue` and `dispatch` return values on-chain (`execId` / the dispatched `Intent`) but the write wrappers resolve to `{ txHash }` only — read the value back with `nextExecId` / `getRecord` if you need it.

#### findWardAgents (registry)

The registry has no `create*Client` factory — it exposes one function plus the row type:

```ts
function findWardAgents(opts: FindWardAgentsOpts): Promise<FindWardAgentsResult>

interface FindWardAgentsOpts {
  publicClient: PublicClient;
  registryAddress: Address;
  onlyActive?: boolean;     // default false (return all)
  pageSize?: number;        // default 200, clamped to [1, 500]
  signal?: AbortSignal;     // aborts BETWEEN pages, not mid-call
}

interface RegistryAgent {
  agent: Address;
  registrar: Address;
  oracle: Address;
  policyId: Hex;
  name: string;
  metadataURI: string;
  tags: readonly string[];
  updatedAt: bigint;
  active: boolean;
}

type FindWardAgentsResult =
  | { ok: true; registryAddress: Address; agents: RegistryAgent[]; totalCount: bigint; pagesRead: number }
  | { ok: false; error: string; agents?: RegistryAgent[]; pagesRead?: number };
```

Walks `agentsPaginated(offset, limit)` until `agentCount()` is exhausted (a view-call walk, not an event scan — Shannon caps `eth_getLogs` at 1000 blocks). Mixed throw/Result contract: **caller mistakes throw** (missing `publicClient` or `registryAddress`, with a `find-ward-agents:` prefix); **runtime failures return `{ ok: false }`** (an `agentCount`/`agentsPaginated` revert, a pagination inconsistency such as an empty page mid-walk, or a `signal` abort) carrying any partial `agents` + `pagesRead` so a degraded-mode UI can render what it got. Callers that promise "never throws" still need a `try/catch` for the programmer-error throws.

Registry ABI functions not covered by `findWardAgents` (call via `WARD_AGENT_REGISTRY_ABI`): `getAgent`, `agents`, `agentList`, `register`, `update`, `setActive`.

### Preflight & gating

#### preflight

```ts
function preflight(args: PreflightArgs): Promise<PreflightResult>

interface PreflightArgs {
  source: PreflightSource;
  intent: EvalIntent;
  spentTodayWei: bigint;
  onWardDecision?: (result: PreflightResult) => void;
  nowSec?: bigint;          // defaults to wall clock; ignored by the chain source
}

interface PreflightResult {
  ok: boolean;
  reason: Hex;              // canonical bytes32 reason code
  reasonText: string;      // decodeReason(reason).description
  source: "chain" | "local" | "spec";
}
```

The front-end gate. It **never throws on a policy rejection** — a rejected intent comes back as `ok: false` with the canonical reason and a decoded description. Programmer errors still throw: an unsupported `source.kind`, or malformed `spec` YAML that reaches `compilePolicy` (which throws). The optional `onWardDecision` callback fires with the final result for telemetry.

`PreflightSource` is a discriminated union of three source kinds:

| `kind` | Shape | Behavior |
|---|---|---|
| `chain` | `{ kind: "chain"; publicClient; oracleAddress; policyId }` | Reads live state via `checkIntent` against the deployed `WardOracle`. Strongest source of truth; one RPC round trip. `nowSec` is irrelevant (the contract uses `block.timestamp`). |
| `local` | `{ kind: "local"; policy: EvalPolicy }` | Evaluates a pre-built `EvalPolicy` synchronously with `evalCheckIntent`. Zero RPC. |
| `spec` | `{ kind: "spec"; yaml: string }` | Compiles the POLICY.md YAML (`compilePolicy` → `evalPolicyFromInput`) then evaluates locally. Pure off-chain; can throw if the YAML is malformed. |

`local` and `spec` use `args.nowSec ?? BigInt(Math.floor(Date.now() / 1000))`.

#### buildIntent

```ts
function buildIntent<const TAbi extends Abi>(args: BuildIntentArgs<TAbi>): EvalIntent

interface BuildIntentArgs<TAbi> {
  abi: TAbi;
  address: Address;
  functionName: ContractFunctionName<TAbi>;
  args?: ContractFunctionArgs<...>;
  value?: bigint;
  requestId: bigint;        // required
  agentId?: bigint;         // default 0n
  promptHash?: Hex;         // default 32 zero bytes
  taskClass?: number;       // default 0
}
```

ABI-encodes the call (`encodeFunctionData`) and derives the `selector` from the matched ABI item, producing an `EvalIntent` ready for `preflight`/`checkIntent`. Throws if `requestId` is `undefined`, or if `functionName` is not a function in the ABI. The returned `data` field is the full calldata; `selector` is its first 4 bytes (the evaluator's `SELECTOR_MISMATCH` check enforces this relationship).

#### withWardPreflight (wallet decorator)

```ts
function withWardPreflight<TWallet extends WalletClient>(
  wallet: TWallet,
  opts: WithWardPreflightOptions,
): TWallet

interface WithWardPreflightOptions {
  source: PreflightSource;
  spentTodayWei: bigint | (() => Promise<bigint>);
  onRejected?: (decision: PreflightResult) => void;
}
```

Wraps a viem `WalletClient` in a `Proxy` that runs `buildIntent` + `preflight` ahead of every `writeContract`. On a rejected decision it calls `onRejected` and **throws** an `Error` whose message is `formatWardUserMessage(reason)`; on `ok` it forwards to the underlying `writeContract`. `writeContract` callers pass an optional `ward: { requestId?, agentId?, promptHash?, taskClass? }` block (defaults `requestId` to `0n`). `sendTransaction` is **not** preflighted — it has no `abi`/`functionName` context, so the proxy logs a `console.warn` and passes the call straight through. The unwrapped client is reachable via the `_underlying` property.

### Evaluation & telemetry

#### policy-eval

The bit-faithful TS port of the on-chain check, for `local`/`spec` preflight.

```ts
function evalPolicyFromInput(input: PolicyInput): EvalPolicy
function evalCheckIntent(policy: EvalPolicy, intent: EvalIntent, spentTodayWei: bigint, nowSec: bigint): { ok: boolean; reason: Hex }
function applyPolicyToObservedCall(policy: PolicyInput, call: ObservedCall, spentTodayWei: bigint): Verdict
```

`evalPolicyFromInput` projects a `PolicyInput` into the normalized `EvalPolicy` lookup form (per-target/per-selector maps, all keys lowercased). It **throws** on a duplicate target or duplicate selector within a target.

`evalCheckIntent` is a pure function (no clock, no I/O — `nowSec` is supplied by the caller) that returns the same `(ok, reason)` shape as the on-chain `WardOracle.checkIntent`. It **throws** (a `RangeError`/`TypeError`) on out-of-domain inputs — values the ABI encoder for `checkIntent` would itself reject (e.g. a `uint256` field that is negative or above `2^256-1`, a non-hex `target`/`selector`/`data`/`promptHash`, a `taskClass` outside `uint8`). Policy-level rejections flow through the `{ ok: false, reason }` return, never a throw. Reason precedence (matches `PolicyLib.validate` + the tier check exactly):

1. `paused` → `PAUSED`
2. `nowSec > expiresAt` → `EXPIRED`
3. `data.length < 4` → `BAD_CALLDATA`
4. `selectorOf(data) != intent.selector` → `SELECTOR_MISMATCH`
5. target not allowed → `TARGET_NOT_ALLOWED`
6. selector not allowed → `SELECTOR_NOT_ALLOWED`
7. `value > valueCapPerCall` → `VALUE_CAP`
8. `spentToday > cap || value > cap - spentToday` → `DAILY_CAP`
9. tier `DELAYED` → `REQUIRES_DELAY`
10. tier `VETO_REQUIRED` → `REQUIRES_VETO`
11. else → `OK`

`applyPolicyToObservedCall` is the older off-chain helper used by the dashboard's watch flow; it takes an `ObservedCall` (`target`, `selector`, `valueWei`, `asker`, `timestampSec`) and returns a `Verdict` (`{ allowed: true; tier: TierName; delaySeconds }` or `{ allowed: false; reason }`) with English `reason` strings. It does not run the tier-queue check or the input-domain guards. Prefer `evalCheckIntent` for new code.

#### reason codes

```ts
const REASON_CODES: {
  OK: Hex;                  // 0x000...0 (empty bytes32)
  PAUSED: Hex;
  EXPIRED: Hex;
  BAD_CALLDATA: Hex;
  SELECTOR_MISMATCH: Hex;
  TARGET_NOT_ALLOWED: Hex;
  SELECTOR_NOT_ALLOWED: Hex;
  VALUE_CAP: Hex;
  DAILY_CAP: Hex;
  REQUIRES_DELAY: Hex;
  REQUIRES_VETO: Hex;
}
type ReasonName = keyof typeof REASON_CODES;

function decodeReason(reason: Hex): { name: ReasonName | "UNKNOWN"; description: string }
```

Each non-`OK` code is the left-aligned, zero-padded `bytes32("NAME")` literal the contract emits (`OK` is the all-zero bytes32 returned on success). `decodeReason` maps a raw reason (case-insensitive) to its name + a one-sentence description; an unrecognized code returns `{ name: "UNKNOWN", description: "..." }` so the decoder is safe in the warm path. There is **no slippage reason code** — `maxSlippageBps` is stored but never checked by `validate`.

#### formatters

```ts
function formatWardDecision(result: PreflightResult, context?: WardDecisionContext): WardDecisionLog
function formatWardUserMessage(reason: Hex): string
```

`formatWardDecision` shapes a `PreflightResult` (plus optional `policyId`/`target`/`selector`/`requestId`/`agentId`/`source` context) into a structured `WardDecisionLog` with `event: "ward.decision"` — `bigint` fields are stringified for JSON safety. `formatWardUserMessage` is `decodeReason(reason).description`.

#### spent-tracker

Pure arithmetic that mirrors the on-chain per-asker, per-UTC-day spend bucket (`block.timestamp / 86400`):

```ts
function utcDayBucket(timestamp: bigint): bigint
function spentTodayFor(entries: SpendEntry[], asker: `0x${string}`, nowTimestamp: bigint): bigint

interface SpendEntry { timestamp: bigint; valueWei: bigint; askerAddress: `0x${string}`; }
```

Feed it the `(timestamp, value, asker)` tuples the contract would have observed to compute a `spentTodayWei` value for `preflight`/`checkIntent` without a per-day on-chain getter. Pure: no clock reads — pass `nowTimestamp` explicitly.

#### event-store

In-memory store of Ward on-chain state, backing the dashboard and TUI. Pure TS (no React/ink). Backfills policy + queue events, hydrates queue record headers, and live-watches both contracts.

```ts
function createEventStore(config: EventStoreConfig): EventStore
```

Key config fields: `publicClient`, `oracleAddress`, `queueAddress`, optional `oracleDeploymentBlock`, `queueLookbackBlocks` (default ~7 days at 1s blocks), `chunkSize` (default 1000 — Shannon caps `eth_getLogs` at 1000 blocks), `eventLogCap` (default 2000), `startBlock`, and an `onProgress` callback. The `EventStore` exposes `init()` (resolves when backfill completes; live watch keeps running), `dispose()`, `subscribe(handler)`, the read accessors (`listPolicies`, `getPolicy`, `listPoliciesByOwner`, `listQueueRecords`, `getQueueRecord`, `listPending`, `listPendingWithExecIds`, `recentEvents`, `cursor`), the `hydrate*` / `hydrate*AndPersist` snapshot-replay methods, and `emitSnapshotUpdated()`. `init()` throws if called after `dispose()`. The store does not persist anything itself — pass a `startBlock` and use the `subscribe`-driven `snapshotUpdated` synthetic to drive your own persistence.

### Queue handoff helpers

Off-chain operator-tooling helpers for deciding how to dispatch a queued record (`src/queue-handoff.ts`):

```ts
function queueTierName(tier: number): QueueHandoffTier
function castSendCommand(address: Address, signature: string, execId: bigint | string): string
function buildQueueHandoffRecommendation(input: QueueHandoffInput): QueueHandoffRecommendation
function abiExposesDispatchQueued(abi: unknown): boolean
function extractAbi(json: unknown): unknown
```

`buildQueueHandoffRecommendation` returns a human-readable summary + a ready-to-run `cast send` command tuned to the record's tier (IMMEDIATE → "should not be queued"; DELAYED → agent `dispatchQueued(uint256)` if the supplied ABI exposes it, else raw queue `dispatch(uint256)`; VETO_REQUIRED → policy-owner-only). `castSendCommand` formats a `cast send <addr> "<sig>" <execId> --private-key $DEPLOYER_PK --rpc-url $SOMNIA_TESTNET_RPC` string. These are pure string/shape helpers — none throws on normal input.

### React hooks reference (`@ward/react`)

`private: true` workspace package; peers are `react`, `viem`, `wagmi`, `@tanstack/react-query`, and `@ward/sdk`. It is a UX layer, not a security boundary — the on-chain oracle still enforces the policy and the agent still reverts on bad calldata. #### useWardGuardedWrite (signature)

```ts
function useWardGuardedWrite(opts: UseWardGuardedWriteOptions): UseWardGuardedWriteReturn

interface UseWardGuardedWriteOptions {
  source: PreflightSource;
  spentTodayWei?: bigint | (() => Promise<bigint>);
}

interface UseWardGuardedWriteReturn {
  write: <const TAbi extends Abi>(args: WriteContractArgs<TAbi>) => Promise<Hex>;
  isPreflightPending: boolean;
  lastDecision: PreflightResult | null;
}
```

Resolves the wagmi config from context, runs `buildIntent` + `preflight`, sets `lastDecision`, and — only on `ok` — calls wagmi's `writeContract`. On a rejected preflight it sets `lastDecision` and **throws `WardPreflightRejectedError`** (a subclass of `Error` carrying a typed `decision: PreflightResult`); the wallet popup never opens. `write` accepts the usual `{ abi, address, functionName, args?, value?, account?, chainId? }` plus an optional `ward: { requestId?, agentId?, promptHash?, taskClass? }` block. `spentTodayWei` defaults to `0n` when omitted.

```tsx
const { write, isPreflightPending, lastDecision } = useWardGuardedWrite({
  source: { kind: "chain", publicClient, oracleAddress: oracle, policyId },
});
await write({ abi: agentAbi, address: agent, functionName: "bump", args: [1n] });
```

The package also exports `createWardGuardedWrite(options)` — the framework-agnostic factory the hook wraps (inject your own `config`, `writeContract`, and `setPending`/`setLastDecision` callbacks) — and `WardPreflightRejectedError`.

#### useWardActionPlan (signature)

```ts
function useWardActionPlan(opts: UseWardActionPlanOptions): UseWardActionPlanReturn
function planWardAction(args: PlanWardActionArgs): Promise<WardActionPlan>

type WardActionPlan =
  | { kind: "write-now"; intent: EvalIntent }
  | { kind: "queue"; tier: "DELAYED" | "VETO_REQUIRED"; delaySeconds: number; intent: EvalIntent }
  | { kind: "reject"; reason: Hex; reasonText: string };

interface UseWardActionPlanReturn {
  plan: WardActionPlan | null;
  isPending: boolean;
  refresh: () => Promise<void>;
  error: Error | null;
}
```

Resolves a prepared `intent` into a `write-now` / `queue` / `reject` plan so the UI can show the tier and delay before the user clicks. It re-evaluates on `refresh()` (and on mount unless `enabled: false`). `queue` plans only resolve for `local` and `spec` sources — those carry the tier maps; `planWardAction` **throws** if it needs a queue tier and `source.kind === "chain"` (the chain source returns only the decision, so look the tier up separately via `tierAndDelay`). The hook surfaces planning failures through `error` rather than crashing the render; `refresh()` itself rejects on failure.

### Vite plugin reference (`@ward/vite-plugin`)

`private: true` workspace package; peers are `vite` and `@ward/sdk`. Default export is a zero-config plugin factory:

```ts
import wardPolicy from "@ward/vite-plugin";
export default defineConfig({ plugins: [wardPolicy()] });
```

It transforms any import whose query string contains `eval-policy` (e.g. `import policy from "./POLICY.md?eval-policy"`) by running `compilePolicy` → `evalPolicyFromInput` at build time and emitting a module whose default export is the evaluated `EvalPolicy` (bigint fields serialized as `…n` literals). The plugin owns no runtime — use the imported object as the `local` source for `preflight` / `useWardGuardedWrite`. It re-exports the `EvalPolicy` type. A malformed policy fails the build (the underlying `compilePolicy`/`serialize` throws during `transform`).

### Throw-vs-Result contract

Exact contract per API. "Throws" means a thrown `Error`/`RangeError`/`TypeError` (a bug in caller code); "Result" means a returned value that encodes the failure.

| API | Policy rejection | Programmer / runtime error |
|---|---|---|
| `compilePolicy` | n/a | **Throws** on any invalid document (no Result envelope). |
| `PolicyBuilder.*` | n/a | **Throws** on invalid tier/cap/delay/bps or `build()` with no targets / unset `expiresAt`. |
| `parseEtherFlexible` | n/a | **Throws** on an unrecognized unit (hint-bearing) or non-numeric input. |
| `OracleClient.publishPolicy` / `updatePolicy` | n/a | **Throws** (simulate reverts before the wallet popup; missing wallet/account throws). |
| `OracleClient.checkIntent` / `tierAndDelay` | Returned in `{ ok, reason }` / `{ tier, delay }` | **Throws** on RPC error. |
| `QueueClient` writes | n/a | **Throws** (missing wallet/account, or on-chain revert). |
| `QueueClient.getRecordHeader` | n/a | **Throws** only if both canonical + legacy decodes fail; otherwise returns the record. |
| `findWardAgents` | n/a | **Result** `{ ok: false }` for RPC reverts / pagination inconsistency / abort; **throws** for missing `publicClient`/`registryAddress`. |
| `preflight` | **Result** `{ ok: false, reason, reasonText }` | **Throws** on unsupported `source.kind` or malformed `spec` YAML. |
| `buildIntent` | n/a | **Throws** if `requestId` undefined or `functionName` not in ABI. |
| `withWardPreflight` (`writeContract`) | **Throws** `Error(formatWardUserMessage(reason))` after `onRejected` | **Throws** on encoding errors. `sendTransaction` is not preflighted. |
| `evalCheckIntent` | **Result** `{ ok: false, reason }` | **Throws** `RangeError`/`TypeError` on out-of-domain inputs. |
| `evalPolicyFromInput` | n/a | **Throws** on duplicate target/selector. |
| `applyPolicyToObservedCall` | **Result** `Verdict` (`{ allowed: false, reason }`) | Pure; does not throw on policy data. |
| `decodeReason` | n/a | Never throws — unknown code returns `{ name: "UNKNOWN" }`. |
| `createEventStore` / `EventStore.init` | n/a | `init()` **throws** if called after `dispose()`; subscriber errors are swallowed. |
| `useWardGuardedWrite.write` | **Throws** `WardPreflightRejectedError` (carries `.decision`) | **Throws** on wagmi `writeContract` errors. |
| `useWardActionPlan` / `planWardAction` | **Result** `{ kind: "reject", ... }` plan | **Throws** when a queue plan is requested with `source.kind === "chain"`; the hook surfaces it via `error`. |
| `wardPolicy()` transform | n/a | **Throws** (fails the build) on a malformed policy file. |
